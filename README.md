# Sustainable Computing Energy Audit Platform

A sophisticated full-stack application for monitoring and analyzing energy consumption, environmental impact, and financial costs in campus computing infrastructure. It combines static manual audits with real-time telemetry to provide actionable intelligence for Responsible Computing.

## Project Overview

The platform provides a comprehensive suite of tools:

- **Audits & Telemetry**: Establish baseline energy use via manual audits and ingest real-time telemetry data from workstations.
- **Dashboard Analytics**: Visualize energy footprints, idle waste, and peak consumption patterns.
- **Predictions & Anomalies**: Forecast future energy costs and carbon emissions, and automatically detect unusual power consumption behavior.
- **Recommendations Engine**: Heuristic-driven engine that provides actionable, prioritized interventions to reduce energy waste.
- **Simulations Engine**: Compare existing hardware against replacements using a rigorous Total Cost of Ownership (TCO) and Carbon ROI mathematical model.
- **Carbon Reporting**: System-wide GHG Protocol-aligned reporting that translates kWh into tangible metrics (like Trees Equivalent).

## Technology Stack

- **Frontend**: Next.js (React 19), App Router, Tailwind CSS v4, shadcn/ui, Recharts
- **Backend & APIs**: Next.js Route Handlers (TypeScript)
- **Database**: PostgreSQL (via `pg` library)

## Setup & Coldstart Guide

Follow these steps to initialize the project and seed it with initial data.

### 1. Prerequisites
- Node.js 18+
- PostgreSQL 12+ running locally

### 2. Database Creation & Schema
Open a PostgreSQL terminal and run:

```sql
CREATE DATABASE energy_db;
CREATE DATABASE workstation_monitor;
```

#### In `energy_db`:
```sql
CREATE TABLE audits (
  audit_id SERIAL PRIMARY KEY,
  audit_name VARCHAR(255) NOT NULL,
  location VARCHAR(255),
  audit_date DATE NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE devices (
  device_id SERIAL PRIMARY KEY,
  audit_id INTEGER NOT NULL REFERENCES audits(audit_id) ON DELETE CASCADE,
  device_class VARCHAR(100) NOT NULL,
  description VARCHAR(255),
  power_rating_watts DECIMAL(10, 2),
  quantity INTEGER,
  hours_per_day DECIMAL(5, 2),
  daily_kwh_total DECIMAL(10, 4) GENERATED ALWAYS AS (
    (power_rating_watts * quantity * hours_per_day) / 1000
  ) STORED,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_devices_audit_id ON devices(audit_id);
```

#### In `workstation_monitor`:
```sql
CREATE TABLE metrics_repository (
  id SERIAL PRIMARY KEY,
  timestamp_utc TIMESTAMP NOT NULL,
  computer_name VARCHAR(255) NOT NULL,
  cpu_percent DECIMAL(5, 2),
  mem_percent_used DECIMAL(5, 2),
  disk_bytes_sec BIGINT,
  inferred_watts DECIMAL(10, 2),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_metrics_timestamp ON metrics_repository(timestamp_utc DESC);
CREATE INDEX idx_metrics_computer ON metrics_repository(computer_name);
```

### 3. Install Dependencies
```bash
npm install
```

### 4. Configure Environment
Create a `.env.local` file in the root directory:
```env
DATABASE_URL=postgresql://postgres:password@localhost:5432/energy_db
```
*(Replace `postgres:password` with your local PostgreSQL credentials. The app primarily connects to `energy_db` and uses the same credentials to access `workstation_monitor`.)*

### 5. Run the Application
```bash
npm run dev
```
Visit http://localhost:3000 to access the platform.

## Coldstart: Getting Data into the System

The platform requires both static (Audit) and dynamic (Telemetry) data to function fully.

1. **Create a Manual Audit:**
   - Go to `http://localhost:3000/audit` > **Manual Audit** tab.
   - Enter an Audit Name and Location.
   - Add devices (e.g., Lighting, Servers) with their respective wattages and daily usage. 
   - Click **Save Audit**.

2. **Upload Telemetry Data:**
   - Go to `http://localhost:3000/audit` > **Upload Telemetry** tab.
   - Paste a sample CSV dataset to jumpstart the analytics engine. Example format:
     ```csv
     timestamp_utc,computer_name,cpu_percent,mem_percent_used,disk_bytes_sec,inferred_watts
     2025-01-15T10:00:00Z,LAB-PC-01,45.2,62.1,1024000,250.5
     2025-01-15T10:05:00Z,LAB-PC-01,48.1,65.3,1048576,265.2
     2025-01-15T10:00:00Z,LAB-PC-02,2.3,42.8,512000,120.3
     2025-01-15T10:05:00Z,LAB-PC-02,1.8,42.8,512000,118.5
     ```
   - Click **Upload Data**.

Once data is ingested, you can navigate to the **Dashboard**, **Predictions**, **Recommendations**, **Simulations**, and **Carbon Reports** pages to explore the insights generated by your metrics. For in-depth explanations of any page's metrics, click the `?` icon in the page header.
